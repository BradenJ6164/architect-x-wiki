NPCs

NPCs (no player character)s in SCP Architect X serve many functions for your foundation. Below is the behavior of each type of NPC. NPCs are meant to be predictable, and have almost no random properties or decisions unless explicitly stated. A lifecycle is a cycle that is repeated through out the life of the NPC. When a lifecycle fails at any action, it immediately jumps to the failure action.

Scientist

Lifecycle:

  • Do Test
  • Do Test
  • Go Eat
  • Research
  • Research
  • Go Sleep

On Fail

  • Roam

Security

Lifecycle:

  • Patrol
  • Patrol
  • Go Eat
  • Patrol
  • Patrol
  • Go Sleep

On Fail

  • Roam

Class D

Class Ds work differently from other NPCs. Class Ds are considered child NPCs. These NPCs often have a parent such as a Scientist or Security NPC. The given parent of a Class D can change at any time. When the Class D is assigned a parent, it's lifecycle is temporarily paused for later resumption. 

Lifecycle:

  • Random:
    • Use Toilet
    • Go Sleep

On Fail:

  • Nothing
