NPCs

NPCs (no player character)s in SCP Architect X serve many functions for your foundation.

NPCs are meant to be predictable, and have almost no random properties or decisions unless explicitly stated.

NPC Behavior

Outlined below is the static behavior of every NPC. An NPC's lifecycle is a loop of actions and decisions an NPC will make during it's life time (from hiring to firing). If a lifecycle fails at any action, it will immediately jump to the failure action and then attempt to restart the lifecycle.

Scientist

Lifecycle:

  • Do Test - Always Succeeds
  • Do Test  - Always Succeeds
  • Go Eat
  • Research - Always Succeeds
  • Research - Always Succeeds
  • Go Sleep

On Fail

  • Roam

Security

Lifecycle:

  • Patrol - Always Succeeds
  • Patrol - Always Succeeds
  • Go Eat
  • Patrol - Always Succeeds
  • Patrol - Always Succeeds
  • Go Sleep

On Fail

  • Roam

Class D

Class Ds work differently from other NPCs. Class Ds are considered child NPCs. These NPCs often have a parent such as a Scientist or Security NPC. The given parent of a Class D can change at any time. When the Class D is assigned a parent, it's lifecycle is temporarily paused for later resumption. 

Lifecycle:

  • Random:
    • Use Toilet
    • Go Sleep

On Fail:

  • Nothing
